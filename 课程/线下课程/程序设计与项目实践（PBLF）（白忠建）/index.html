    程序设计与项目实践（PBLF）（白忠建）

    大型程序设计规范

    一、编程规范

    常量名

    要养成使用符号常量的变成习惯

    在主程序中使用常量时，最好通过宏定义用符号表示常量

    • 全大写，单词之间用下划线连接

    如：

    #define MAX_NUM_OF_STUDENTS 1024

    变量名

    • 小驼峰表示法：小写开头，后面的每个单词的首字母大写，单词之间不用下划线。

    如：

    int numOfStudents;
    • 指针变量加前缀p（匈牙利命名法）

    如：

    char* pChar;

    自定义类型名

    • 大驼峰表示法：每个单词的首字母大写，单词之间不用下划线

    如：

    typedef struct abc {
  int a;
  char b;
  double c;
} LinkedList;

    函数名

    • 成员函数：小驼峰表示法。
    • 非成员函数：大驼峰表示法。
    • 带参数的宏：与函数同。
    • 函数名应做到见名即知函数的功能。
    • 库函数中的函数名未遵守现代的编程规范是由于历史的原因。

    函数注释

    /* 函数的功能
* 输入参数的注释
* 输出参数的注释
* 返回值的注释
* 使用此函数的注意事项
*

    源文件的格式

    1. 文件包含
    2. 常量定义
    3. 类型定义
    4. 全局变量（最好没有）
    5. 函数声明
    6. 函数实现
    7. 主函数

    二、结构化设计

    把大程序分解为小模块（分而治之）（模块化）

    高内聚（一个函数只做一件事，函数内部没有多余的东西），松耦合（函数与函数之间没有太多联系，函数的输入输出都通过参数传递（I/O函数除外））

    三、多文件程序

    源文件

    大型程序由多个源文件（.c文件）组成

    只能有一个源文件中有主函数

    多个源文件的好处：

    • 把相关的函数和类型放入一个文件中使程序的结构更清晰。
    • 便于代码复用。
    • 便于多个程序员分工合作。
    • 每个源文件可以单独编译，节省编译的时间。

    头文件

    一个源文件如何使用另一个源文件里的函数呢？

    用 #include "XXX.h"

    头文件中的内容

    常量定义
类型定义
全局变量（慎用）
函数原型
详细的注释（重要）

    对于函数来说：

    （多个）源文件的作用：函数的实现

    （多个）头文件的作用：列出函数原型

    构建多文件程序

    步骤：

    1. 编译：每个源文件单独编译（不需要编译头文件）。一个源文件产生一个目标文件，在UNIX中为.O，在Windows中为.obj
    2. 链接：把编译产生的目标文件以及库函数链接在一起，形成可执行文件。

    步骤的实现（以gcc命令为例）

    gcc -o calc calc.c token.c stack.c

    首先把三个源文件分别编译成目标文件

    然后链接器把这些目标文件合在一起。

    选项-o表明我们希望的可执行文件名是calc
