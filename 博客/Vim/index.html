    Vim

    vimtutor教程

    使用vim打开文件

    在terminal下输入vim <filename>以使用vim编辑器打开<filename>

    退出vim

    在normal模式下输入:q!<enter>

    这会在不保存更改的情况下强制退出vim

    文本编辑

    删除

    在normal模式下，按x键来删除光标所在位置的字符

    删除类命令
    1. 在normal模式下，输入dw可以从光标处删除至一个单词的末尾
    2. 在normal模式下，输入d$可以从当前光标删除到行末
    3. 删除整行：鉴于整行删除的高频度，Vi 的设计者决定要简化整行删除操作，您仅需要在同一行上击打dd就可以删除掉光标所在的整行了。

    你还可以：输入2dd删除两行。

    插入：在光标前插入文本

    在normal模式下，按i键来插入文本

    这会使vim进入insert模式，编辑完成之后，按<esc>键回到normal模式

    添加：在一行后添加文本

    在normal模式下，按a键来添加文本

    这会使vim进入insert模式，编辑完成之后，按<esc>键回到normal模式

    提取和合并文件

    要向当前文件中插入另外的文件的内容，请输入:r FILENAME

    您所提取进来的文件将从光标所在位置处开始置入

    打开类命令

    输入o将在光标的下方打开新的一行并进入插入模式

    输入大写O将在光标的上方打开新的一行并进入插入模式

    附加类命令

    输入a将可在光标之后插入文本

    提示：a、i 和 A 都会带您进入插入模式，惟一的区别在于字符插入的位置。

    复制与粘贴

    使用操作符y复制文本，使用p粘贴文本

    可以现在visual模式下选择之后再进行复制

    剪切

    在visual模式下选择后按d，粘贴时按p

    置入

    输入p将最后一次删除的内容置入光标之后。

    1. 输入 dd 将该行删除，这样会将该行保存到 Vim 的一个寄存器中。

    2. 接着将光标移动到 c) 一行，即准备置入的位置的上方。记住：是上方哦。

    3. 然后在正常模式下(键进入)输入 p 将该行粘贴置入。

    替换

    输入r和一个字符替换光标所在位置的字符

    输入大写的R可连续替换多个字符

    全局替换

    输入s/old/new <enter>可以替换光标所在行中第一个old为new

    输入s/old/new/g <enter>可以替换整行中的old为new

    输入:#,#s/old/new/g其中#,#代表的是替换操作的若干行中首尾两行的行号。

    输入:%s/old/new/g则是替换整个文件中的每个匹配串。

    输入:%s/old/new/gc会找到整个文件中的每个匹配串，并且对每个匹配串提示是否进行替换。

    更改

    更改类操作符是c

    如：

    要改变文本直到一个单词的末尾，请输入ce

    更改类操作符的工作方式跟删除类是一致的。操作格式是：

    c    [number]   motion

    保存与退出

    依次按:wq键以保存文件并退出

    特别提示：按下 键会带您回到正常模式或者撤消一个不想输入或部分完整 的命令。

    将对文件的改动保存到空文件中

    请输入:w <FILENAME>

    此操作需要shell所处目录中存在空文件

    删除文件:!rm <FILEMANE>

    具有选择性的保存命令

    要保存文件的部分内容，请输入v motion :w FILENAME

    选择文件后输入:w <FILENAME>

    这时 Vim 会把选中的行写入到以 TEST 命名的文件中去

    操作符

    • d - 删除
    • c - 更改

    指针

    移动指针

   < h       l >               l 的键位于右边，每次按下就会向右移动。
         j                     j 键看起来很象一支尖端方向朝下的箭头。

    整块移动指针

    • w - 从当前光标当前位置直到下一个单词起始处，不包括它的第一个字符。
    • e - 从当前光标当前位置直到单词末尾，包括最后一个字符。
    • $ - 从当前光标当前位置直到当前行末。

    许多改变文本的命令都由一个操作符和一个动作构成。

    如：

    使用删除操作符 d的删除命令的格式如下：

        d   motion

    其中： d - 删除操作符。 motion - 操作符的操作对象(在下面列出)。

    因此输入 de 会从当前光标位置删除到单词末尾。

    特别提示：

    对于勇于探索者，请在正常模式下面仅按代表相应动作的键而不使用操作符，您将看到光标的移动正如上面的对象列表所代表的一样。

    定位及文件状态

    • 输入CTRL-G显示当前编辑文件中当前光标所在行位置以及文件状态信息
    • 输入大写G则直接跳转到文件中的某一指定行
    • 输入gg可以使得当前光标直接跳转到文件第一行。
    • 输入您曾停留的行号，然后输入大写G跳转到相应行

    查找配对括号

    输入%可以查找配对的括号)、]、}

    把光标放在想要查找的括号处然后按%

    计数

    1. 在动作前输入数字n会使它重复n次

    如：

    • 输入 2w 使光标向前移动两个单词。
    • 输入 3e 使光标向前移动到第三个单词的末尾。

    • 输入 0 (数字零) 移动光标到行首。

    • 使用操作符时输入数字n可以使它重复n次

    d number(数字) motion

    如：

    • 输入d2w以删除两个单词。

    撤销

    输入u来撤消最后执行的命令，输入U来撤消对整行的修改。

    输入⌃R(先按下⌃键不放开，接着按R键)，这样就可以重做被撤消的命令

    搜索

    在normal模式下输入/<想要搜索的文字>即可在当前文件中查找该文字

    要查找同上一次的字符串，只需要按n键。要向相反方向查找同上一次的字符串，请输入大写N即可。

    如果您想逆向查找字符串，请使用 ? 代替 / 进行。

    要回到您之前的位置按 CTRL-O (按住 Ctrl 键不放同时按下字母 o)。重复按可以回退更多步。CTRL-I 会跳转到较新的位置。

    执行外部命令

    输入:!然后紧接着输入一个外部命令再按<enter>可以执行该外部的shell命令

    获取帮助

    Vim 拥有一个细致全面的在线帮助系统。要启动该帮助系统，请选择如下三种方法之一：

        - 按下 <HELP> 键 (如果键盘上有的话)
    - 按下 <F1> 键 (如果键盘上有的话)
    - 输入  :help <回车>

    请阅读帮助窗口中的文字以了解帮助是如何工作的。

    输入 CTRL-W CTRL-W 可以使您在窗口之间跳转。

    输入 :q <回车> 可以关闭帮助窗口。

    提供一个正确的参数给":help"命令，您可以找到关于该主题的帮助。请试验以下参数(可别忘了按回车键哦)：

        :help w
    :help c_CTRL-D
    :help insert-index
    :help user-manual

    补全功能

    使用 CTRL-D 和 <TAB> 可以进行命令行补全

    1. 请确保 Vim 不是在以兼容模式运行： :set nocp

    2. 查看一下当前目录下已经存在哪些文件，输入： :!ls 或者 :!dir

    3. 现在输入一个目录的起始部分，例如输入： :e

    4. 接着按 CTRL-D 键，Vim 会显示以 e 开始的命令的列表。

    5. 然后按 键，Vim 会补全命令为 :edit 。

    6. 现在添加一个空格，以及一个已有文件的文件名的起始部分，例如： :edit FIL

    7. 接着按 键，Vim 会补全文件名(如果它是惟一匹配的)。

    提示：补全对于许多命令都有效。您只需尝试按 CTRL-D 和 <TAB>。它对于 :help 命令非常有用。

    学习vim过程中遇到的一些问题

    关于mac上vscode的vim光标无法通过长按hjkl持续移动的问题

    这是mac系统自带的功能，即长按按键可以输入带音调的字母，现在我们就只在vscode下关闭这个功能：

    # To disable the Apple press and hold for VSCode only
defaults write com.microsoft.VSCode ApplePressAndHoldEnabled -bool false

    要想恢复：

    # To re-enable, run this command in a terminal
defaults write com.microsoft.VSCode ApplePressAndHoldEnabled -bool true

    关于vim快捷键与vscode快捷键冲突的问题

    使用 Vim 插件提供的快捷键映射功能，把原C-r改成<leader>+<C-r>

    在setting.json中添加如下命令

    "vim.normalModeKeyBindings": [
        {
            "before": [
                "<leader>",
                "<C-r>",
            ],
            "commands": [
                "workbench.action.openRecent",
            ],
        },
    ],

    简单说下上面配置的含义：

    当在 Normal 下按键 + 时，会调用并执行 VS Code 内部命令workbench.action.openRecent，这个命令其实就是VS Code 原生快捷键 触发的内部命令。

    这样即不会影响Vim原生快捷键，也不用修改VS Code原生的快捷键。只是当你处于文本编辑工作时，如果需要操作VS Code原生快捷键需要先按一下键而已。注意，这里我特意强调了下“文本编辑”，因为当你处于其他操作时Vim模拟器的快捷键是不生效的，也就是说，你不需要按键直接使用VS Code快捷键就可以。
